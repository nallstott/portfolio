Role Art Director / Designer / Developer

As new Dungeons and Dragons players, my friends and I would get frustrated at how difficult it was choosing spells for our characters when we'd start out or level up. The Player's Handbook wasn't exactly user friendly, so I created a clickable version.

Class Spell {

All the Spells

Arguably, the largest challenge of a website cataloging every spell in the Player's Handbook was in structuring how that information would be retrieved and delivered. Thus, a massive JavaScript file was created, featuring 4084 lines of spells.

Retrieval

Show Me Spells

Using spells.js, each page on the site used the individual spell's JavaScript object to retrieve every bit of information regarding the spell.

Breaking the Law

The SRD

While Wizards of the Coast, the parent company of Dungeons & Dragons, gives away a lot of content for free, they don't feel that way about all of it. When I posted the site to Reddit, that's when I learned about the SRD, a document of everything WOTC gives to the public for free. This did not include many of my spells. This triggered an entire restructuring of the entire site into PHP.

Bending the Law

Getting Around the SRD

Certainly, I have no issue respecting intellectual property laws. But hey, I'm not here to make money, I'm here so my friends can look up spells, so for private use, I threw a password on the site. Since cookies are a living hell in JS, I used PHP to store a session specific password which unlocked all Non-SRD content on the site including many spells and class specefic features like Druid Circles. This also extended to the site's search function. To access this content and see the difference, go ahead and type "ass" into that line and hit return.
